# UX Opportunity Hunting Checklist

Before dismissing any opportunity, run this checklist. Goal: ensure every potential UX opportunity is fully qualified — not assumed away.

## 1. Opportunity Basics (Forensics)

Have you reviewed the opportunity in:
- [ ] EVO (Caylent platform)
- [ ] Salesforce
- [ ] Proposal / ROM (if available)

Do you understand:
- [ ] The business problem being solved?
- [ ] The project goals?
- [ ] The current stage (active / proposal / lost)?

If lost or dropped, do you know why?
- [ ] Cost
- [ ] Unclear MVP
- [ ] Scope gaps
- [ ] "Client has UX"
- [ ] Technical-only framing

## 2. User & Workflow Signals

Check for language signals, project signals, risk signals, and behavior signals. See `signal-patterns.md` for the full list.

Rule of thumb: If 2+ signals are present → this is worth hunting.

## 3. Delivery Risk Check

Without UX involvement, is there risk of:
- [ ] Unclear MVP definition
- [ ] Edge cases missed
- [ ] Error states undefined
- [ ] Confusing user flows
- [ ] Rework in later phases
- [ ] Adoption or trust issues (especially AI)
- [ ] Engineering making UX decisions by default

**If 2+ boxes checked → escalate UX.**

## 4. "Client Has UX" Validation

Clarify what this actually means:
- [ ] Do they have product designers or marketing/UI designers?
- [ ] Are they defining workflows or only visuals?
- [ ] Are they designing AI interactions and trust patterns?
- [ ] Are they validating with users?
- [ ] Do they own MVP definition?

If gaps exist → UX can add value.

## 5. What Was Missed or Dropped

- [ ] Was UX originally in ROM but removed?
- [ ] Was MVP unclear or undefined?
- [ ] Were workflows or personas missing?
- [ ] Were operational scenarios ignored?
- [ ] Was adoption strategy absent?

These are UX entry points.

## 6. Right-size the UX Role

Smallest valuable UX involvement:
- [ ] Discovery sprint (2–4 weeks)
- [ ] Workflow mapping
- [ ] AI interaction design
- [ ] Prototype for clarity
- [ ] Design guardrails for engineering
- [ ] Design system alignment
- [ ] Usability/risk review

Define:
- Duration: ________
- Outcome: ________________

Position as: "Reduce rework + protect delivery + increase adoption"

## 7. Sales Intercept Readiness

Can you clearly state:
- [ ] The risk without UX
- [ ] The value of UX
- [ ] The smallest recommended UX scope
- [ ] Who to contact (AE / CSA / PM)

**Arm yourself with:**
1. **The Risk:** Without UX, this project risks rework, unclear MVP, and low adoption.
2. **The Opportunity:** A small UX investment now can define workflows and reduce downstream cost.
3. **The Ask:** Include a targeted UX role (not a massive scope).

**One-liner:**
"This project has user-facing workflows and AI elements. Without UX, we risk unclear MVP and rework. A short UX engagement could define workflows and reduce delivery risk."

## 8. Log It

- [ ] Opportunity added to VRU tracker
- [ ] Status updated
- [ ] Notes captured
- [ ] Next action assigned
- [ ] Owner identified

## Hunter Rules

- Don't assume → investigate
- Don't dismiss → qualify
- Don't overscope → right-size
- Don't wait → intercept early
- Don't hunt alone → document it
