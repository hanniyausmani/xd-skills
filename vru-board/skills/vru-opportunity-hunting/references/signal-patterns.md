# VRU Signal Patterns

Full reference for the UX opportunity hunting signal detection system.

## Language Signals — What People Say

When you hear any of these phrases in Slack, Gong calls, or conversations, pause and investigate:

- "We're not totally sure what the MVP is yet"
- "We'll figure out the UI later"
- "Engineering can handle the UX"
- "The client already has designers"
- "We just need something pretty"
- "It's mostly backend… but there will be a dashboard/portal"
- "Phase 2 will focus on usability"
- "We don't want to slow things down with process"

**Translation Rule:**
- "This is a technical project" → "This is a UX risk project."
- "They already have UX" → "They may still need workflow, AI, or adoption design."

## Project Signals — What's in Scope

These project characteristics are strong UX indicators. Any combination of 2+ signals = hunt:

- AI / GenAI features
- Dashboards or portals
- Internal tools or admin workflows
- Automation replacing manual work
- Multiple data sources
- Complex rules or decision logic
- User-facing configuration or setup
- Any form entry or reporting
- New workflows vs. existing ones
- Web App Developer is staffed
- Product Manager is staffed

## Risk Signals — What's Missing

When you notice these gaps, UX is needed:

- MVP not clearly defined
- No user personas identified
- No workflow diagrams
- No error states discussed
- No adoption plan
- No success metrics
- No validation with users
- Proposal is purely technical
- UX was in ROM but removed

## Behavior Signals — How Teams Are Acting

- Proposal is hard to find or incomplete
- Opportunity data is fragmented (Salesforce vs EVO vs docs)
- Decisions made without user context
- UX discussed late in the cycle
- "We already sent the proposal"
- No designer listed on the team
- PM or architect expressing uncertainty about user needs

## High-Value Project Types

### GenAI / AI Projects (~30-35% of CNA opportunities)
- GenAI POC / GenAI Project / GenAI Strategy
- GenAI Knowledge Base / Chatbot POC
- GenAI Build Out / GenAI Change Order (CO)
- AI Voicebot / Voice bot POV
- **Always have a user interface component — always relevant for UX**

### Data & Analytics (CDE, ~30% of UX opportunities)
- Data Lake / Data Platform POC
- Quicksight / Data Visualization / Reporting
- Analytics Platform / Enterprise Search / Semantic Search

### Application Development (CNA, ~60% of UX work)
- App Dev / App Modernization / App Assessment
- Mobile App / Web App / Portal Development
- Frontend Dev / UI CO / Designer engagements
- MVP QA/QC / Technical Production

### Common Evolution Pattern
GenAI Strategy → GenAI POC → GenAI Build Out → Production
(each phase is a new UX entry point)

## Practice Area Distribution for UX

- **CNA (Cloud Native App Dev):** ~60% — application dev, modernization
- **CDE (Cloud Data Engineering):** ~30% — dashboards, analytics, visualization
- **CAE (Cloud Architecture & Engineering):** ~5% — infrastructure with user-facing components
- **CCE (Center for Customer Evolution):** ~5% — strategy and workshops
